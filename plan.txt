Aqui está um planeamento estruturado para criar esse organizador inteligente de ficheiros. O nome de código para este projeto pode ser SmartSort.

1. Visão Geral e Objetivos
   Criar um serviço local (daemon/script) que monitoriza um ou mais diretórios específicos (ex: Downloads, Ambiente de Trabalho). Quando um novo ficheiro é descarregado ou criado, o sistema deteta-o, analisa o seu conteúdo ou metadados, classifica-o e move-o para a pasta de destino correta, aplicando uma convenção de nomenclatura padronizada.

2. Principais Funcionalidades
   Monitorização em Tempo Real: Deteção imediata de novos ficheiros nas pastas alvo usando eventos do sistema operativo (sem sobrecarregar a CPU).
   Extração de Conteúdo: Capacidade de ler PDFs, Documentos Word e, opcionalmente, extrair texto de Imagens (OCR).
   Classificação Inteligente:
   Nível 1: Regras estáticas baseadas em nomes, extensões ou palavras-chave no texto.
   Nível 2: Utilização de uma API de IA (ex: Google Gemini) para analisar o conteúdo do ficheiro e inferir o seu contexto/projeto automaticamente.
   Organização e Renomeação: Criação automática de árvores de diretórios (se necessário) e renomeação de ficheiros para um formato consistente (ex: 2026-03-02_Fatura_Eletricidade.pdf).
   Gestão de Estado: Lidar com ficheiros que ainda estão a ser descarregados (ficheiros temporários de browsers) antes de os tentar processar.
3. Stack Tecnológica Recomendada
   Python é a escolha ideal pela sua vasta ecologia de bibliotecas de manipulação de ficheiros, integração com IA e scripts de automação.

Linguagem: Python 3.10+
Monitorização de Ficheiros: watchdog (para reagir a eventos do File System).
Processamento de Ficheiros:
PDFs: pypdf ou pdfplumber (para extrair texto de faturas, relatórios).
Imagens (OCR): pytesseract (interface para o Tesseract OCR).
Metadados genéricos: os e pathlib (standard library).
Inteligência Artificial (Opcional, mas recomendado): API oficial da Google, google-genai, para enviar o texto extraído e pedir à IA que devolva uma classificação (ex: "Isto pertence à categoria Impostos ou ao Projeto X?").
Configuração: Ficheiro config.json ou config.yaml para o utilizador definir quais as pastas a vigiar, quais as categorias de destino e regras de renomeação. 4. Arquitetura do Sistema
A arquitetura deve ser dividida em módulos para facilitar a expansão:

Watcher (Observador): Ouve eventos na(s) pasta(s) configurada(s). Ao detetar a criação/fecho de um ficheiro, adiciona-o a uma Fila de Processamento (Queue).
Dispatcher (Despachante): Retira ficheiros da fila. Tenta abrir o ficheiro. Nota importante: Tem de verificar se o browser já terminou o download (verificando extensões como .crdownload ou tentando obter um lock).
Extractor (Extrator de Contexto): Conforme a extensão (.pdf, .jpg, .txt), extrai o texto principal ou os metadados.
Brain (Classificador): Analisa o texto extraído contra as regras do config.yaml ou interroga a API da IA para decidir qual é o Folder de destino e o novo Nome.
Mover (Executor): Cria a pasta de destino (se não existir), move e renomeia o ficheiro, e regista a ação num ficheiro de .log (para caso precises de saber onde foi parar um ficheiro). 5. Fases de Implementação (Roadmap)
Fase 1: Estrutura Base e Monitorização (Onde recomendo começarmos)
Configurar o projeto Node.js atual ou inicializar um novo projeto Python (visto que o diretório atual tem ficheiros node/docker, mas Python seria melhor para isto).
Implementar o watchdog que apenas deteta novos ficheiros numa pasta de Downloads_Teste e imprime o nome deles no terminal.
Fase 2: Classificação por Regras Simples e Ação
Ler a extensão do ficheiro.
Regra simples: Se for .pdf, move para a pasta Documentos. Se for .jpg, move para Imagens.
Implementar a lógica de mover e renomear com segurança.
Fase 3: Extração de Texto
Integrar a leitura de texto de dentro de ficheiros PDF (ex: procurar pela palavra "Fatura" ou "Invoice" dentro do documento).
Fase 4: Integração de IA
Sempre que um ficheiro é complexo, extrair o texto, enviar como prompt para o Gemini: "Lê o texto deste documento. Indica a que categoria pertence: [Trabalho, Pessoal, Finanças, Setup_PC]. Responde apenas com a categoria."
Fase 5: Daemonização
Transformar o script num serviço de fundo que arranca automaticamente quando ligas o computador.
Como reparo que estás num ambiente Linux e tens um package-lock.json e dockerfile no teu diretório atual (/workspaces/SmartSort/), preferes que desenvolvamos isto em Python (que é o padrão para este tipo de automação pela facilidade com AI e ficheiros locais) ou em Node.js/TypeScript (visto que pareces ter um ambiente Node configurado)?

Se quiseres, posso começar já a inicializar o projeto com os ficheiros base e o script de monitorização na linguagem da tua preferência!
